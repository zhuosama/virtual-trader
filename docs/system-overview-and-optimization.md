## 一、当前系统架构

### 1. 核心组件

```
虚拟盘系统架构：

数据层：
├─ 账户数据（accounts/）
│  ├─ main.json：主账户状态
│  └─ lab.json：实验账户状态
├─ 策略配置（strategies/）
│  ├─ active.json：当前策略
│  └─ changelog.json：迭代历史
├─ 交易记录（trades/）
│  └─ YYYY-MM/YYYY-MM-DD.json
├─ 分析报告（reports/）
│  ├─ daily/：每日报告
│  └─ weekly/：周报
├─ 市场数据（market-data/）
│  └─ watchlist.json：关注池
└─ 运维数据（backups/, scripts/）
   ├─ backups/：备份目录
   └─ scripts/：运维脚本

执行层：
├─ Cron任务（5个核心任务）
│  ├─ 盘前分析（08:10北京）
│  ├─ 盘后复盘（15:30北京）
│  ├─ 每日财经（08:00北京）
│  ├─ 网格交易（08:00北京）
│  └─ 周末复盘（周六08:00北京）
├─ 策略引擎
│  ├─ 主策略：价值趋势混合
│  └─ 实验策略：板块轮动动量
└─ 数据获取
   ├─ 腾讯财经API（主要）
   └─ 备用数据源

监控层：
├─ 完整性检查（check_integrity.sh）
├─ 备份系统（backup.sh）
├─ 恢复机制（restore_latest.sh）
└─ 异常预警（待实现）
```

### 2. 任务流程

**每日流程**：
```
08:00 北京 → 每日财经报告（Cron自动）
08:10 北京 → 盘前分析（Cron自动）
09:30-11:30 → 交易时段（策略引擎）
13:00-15:00 → 交易时段（策略引擎）
15:30 北京 → 盘后复盘（Cron自动）
收盘后 → 备份数据（建议）
```

**每周流程**：
```
每日 → 盘前分析、盘后复盘
周六08:00 → 周末深度复盘（Cron自动）
周一 → 检查系统完整性
周五 → 备份数据
```

### 1. 主账户策略 v1.0.4

**策略名称**：价值趋势混合策略  
**版本历史**：v1.0.0 → v1.0.1 → v1.0.2 → v1.0.3 → v1.0.4

**核心逻辑**：
```python
主策略逻辑 = {
    "选股标准": {
        "基本面": "ROE>12%或股息率>3%",
        "趋势确认": "MA5>MA20或MA20>MA60",
        "估值合理": "PE历史分位<80%",
        "流动性": "日成交额>3亿"
    },
    "仓位管理": {
        "单笔最大": "10%",
        "建仓节奏": "分3批，每批间隔3-5天，单批≤8%",
        "总仓位上限": "80%",
        "底仓目标": "60%"
    },
    "止盈止损": {
        "止盈": "涨幅15%或PE>历史80%分位",
        "止损": "跌幅7-8%或基本面恶化",
        "时间止损": "20个交易日无涨幅减仓50%"
    },
    "商品价格维度": {
        "适用板块": ["煤炭", "有色", "钢铁", "化工", "建材"],
        "分析框架": "基本面(40%)+市场面(30%)+商品价格(30%)",
        "动态调整": {
            "煤价-10%": "仓位-20%",
            "成长风格": "仓位-10%",
            "股息率<4.5%": "考虑止损",
            "三维共振向上": "仓位可至12%"
        }
    }
}
```

**迭代经验**：
1. 格力电器单日大额买入导致仓位超限 → 分批建仓
2. 茅台高位建仓教训 → 趋势确认更严格
3. 中国神华分析 → 增加商品价格维度
4. 周期股管理 → 三维一体分析框架

### 2. 实验账户策略 v1.0.4

**策略名称**：板块轮动+动量突破策略  
**版本历史**：v1.0.0 → v1.0.1 → v1.0.2 → v1.0.3 → v1.0.4

**核心逻辑**：
```python
实验策略逻辑 = {
    "入场条件": {
        "动量突破": "放量突破20日高点",
        "技术确认": "MACD金叉",
        "板块联动": "所属概念板块当日涨幅>2%",
        "量能过滤": "量比>1.4"
    },
    "出场条件": {
        "止盈": "涨幅达12%或RSI超买(>75)",
        "止损": "跌幅达5.5-6%或跌破5日均线",
        "时间止损": "持有5个交易日无涨幅则清仓"
    },
    "仓位管理": {
        "单笔最大": "20%",
        "初始仓位": "15%",
        "总仓位上限": "90%",
        "板块最大": "40%"
    }
}
```

**迭代经验**：
1. 量比放宽至1.3增加假突破风险 → 收紧至1.4
2. trailing stop 6%未能及时止损 → 收紧至5.5%
3. 阳光电源假突破教训 → 量比过滤不能完全避免
4. 立讯精密成功案例 → 动量策略在趋势行情中有效
